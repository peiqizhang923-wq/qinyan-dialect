# 秦言三韵 · 功能模块详细介绍

> 每个模块约 350 字 | 引流逻辑 + AI 应用 + 技术体现

## 一、首页总览

**引流逻辑：** 首页是整站流量的分发中枢。六个功能卡片分为"AI 工具"和"趣味互动"两排平级呈现——不让用户产生"这是个技术工具站"的距离感，而是"什么东西都有、先逛逛看"的探索欲。每个卡片只放图标、标题和一句话功能描述，降低认知负担，一秒内用户就能判断自己想去哪。页面底部嵌入了随机老陕谚语模块，每次刷新展示不同谚语，给回头客制造微小的新鲜刺激，降低"来过就不想再来"的概率。左侧固定侧边栏展示全部十个功能入口，当前所在页面高亮标记，确保用户在任何子页面都不会迷路，整个站内跳转距离不超过一次点击。

**技术体现：** 侧边栏导航采用事件委托 + `fadeTo()` 过渡函数，`cubic-bezier(0.22,1,0.36,1)` 仿书页轻翻的视觉节奏。页面加载后自动触发 `preloadPages()` 预加载全部 10 个子页面的 HTML 到 XHR 缓存，后续导航从缓存直接渲染，实现接近 SPA 的切换速度。所有状态通过 `STATE` 对象统一管理、`localStorage` 自动持久化，跨页面共享无需后端。

## 二、乡音翻译机

**引流逻辑：** 翻译机是整站最核心的获客引擎——它的引流模式是"免费试用 → 结果可分享 → 社交裂变"。用户输入自己的日常用语，AI 实时转译成地道的关中、陕北或陕南方言。这个结果天然具有社交传播属性——"用陕西话怎么夸人""我的名字用陕北话怎么说""今天心情嫽咋咧"——用户截图分享到社交媒体的意愿非常高。三种方言并列标签而非下拉菜单，用户会自发切换对比"同一句话在不同地方怎么说"，单次访问就能产生多次体验。右侧的方言词例卡不输入也能逛——关中、陕北、陕南各一列，降低首次使用门槛。你逛着逛着看到"咥"字好奇，顺手就在输入框打一句试试，转化路径极短。

**AI 应用：** 底层调用火山引擎豆包大模型，通过角色扮演 System Prompt 激活方言生成——关中 Prompt 设定为"老西安本地人、市井口语、句式简短"，陕北设定为"黄土高原老乡、豪迈质朴"，陕南设定为"陕南居民、轻柔温婉"。temperature=0.2 低温度参数保证输出一致性和可控性。AI 不仅做词汇替换，还能理解语境的正式/随意程度，输出对应风格，这是传统词表替换做不到的。

**技术体现：** `callAI()` 统一封装 OpenAI 兼容的 chat completions 调用，Bearer Token 鉴权，`max_tokens=150` 保证响应速度。文本在发给 TTS 之前经过双重预处理——JS 端和 Python 端分别对 12 种方言语气词后自动插入逗号，超过 18 字的长句在动词/方位词前智能断句。语音播放通过 `speakText()` 引擎路由——优先 Edge-TTS 微软陕西话神经语音（五档语速音调参数），失败自动切换讯飞 WebSocket，再失败走浏览器 Web Speech 离线兜底，三级级联对用户完全透明。

## 三、大唐雅言

**引流逻辑：** 大唐雅言定位文旅场景的流量入口。内置 11 个西安标志性名胜——大雁塔、钟楼、城墙、回民街、兵马俑、华清池等——以可点击标签形式排列。游客在景区现场打开页面，点击所在景点名称，AI 自动生成一句古典长安风格的雅句。这个场景的传播链条非常明确：游客生成一句古风文案 → 截图发朋友圈/小红书 → 朋友看到问"什么工具生成的好有文化" → 扫码或搜索进入。每一个景区本质上都是免费的地推点位。即使不在现场，用户看到"大雁塔""回民街"这些熟悉的名称也会产生点击冲动。此外，输出以竖排版呈现，视觉上就很有分享价值。

**AI 应用：** System Prompt 设定为"将现代文案转化为贴合大唐长安风格的古典文雅短句"，temperature 设为 0.2，max_tokens 为 60 字——古文贵精不贵多。AI 调用与翻译机共享同一 `callAI()` 函数，仅 Prompt 和输出长度不同。这种"多 Prompt 单引擎"的架构使得增加新功能只需新增一段提示词，无需训练新模型或调整管线。

**技术体现：** 按钮点击触发 `SFX.stamp()` 官印音效——Web Audio API 合成，Sine 120→40Hz 频率滑降叠加 Triangle 60Hz 辅振荡器，模拟混响。结果采用水墨晕染动画——`filter: blur(5px)→0` + `opacity 0→1`，0.8 秒过渡，模拟墨在宣纸上渗透扩散的物理感。CSS `writing-mode: vertical-rl` 实现竖排版，配合等宽楷体模拟简牍阅读顺序。11 个景点标签为横向滚动容器，支持触屏滑动和鼠标滚轮。

## 四、秦字解码局

**引流逻辑：** 解码局截获的是搜索引擎长尾流量。当用户在百度或微信搜索"咥是什么意思""嫽怎么读""陕西方言有哪些"，解码局如果能通过 SEO 被检索到，就是精准的方言兴趣人群入口。页面顶部是搜索框，下方排列了 12 个高频方言字词的示例标签——"咥""嫽""谝""受活""碎娃""麻达"——用户无需打字，点一下标签就能看到完整解析。输出结构化为四维信息：发音（含声调）、释义（标准语义）、地域归属（关中/陕北/陕南）、民俗典故（历史背景或使用场景）。用户往往从好奇一个字开始，看完典故后忍不住再点下一个——这种"知识链式浏览"天然拉长停留时间。

**AI 应用：** 调用豆包大模型，System Prompt 设定为"对输入的陕西方言词汇进行解码，按发音、释义、地域、典故四段格式输出"，max_tokens 120 字。AI 不仅给出词典式解释，还能补充民俗背景——比如查询"咥"，AI 会讲到《诗经》中已有"佼人嫽兮"的用例，以及关中人对"咥一碗面"的文化情感。这种有深度的内容才是驱动用户停留和分享的核心。

**技术体现：** `pickExample()` 函数实现"点示例 → 填输入框 → 自动触发查询"的一键式体验。搜索结果通过 `showModal()` 模态框展示，支持一键复制文本，适配分享场景。24 条方言词汇种子数据预置在 SQLite 数据库 `dialect_vocabulary` 表中，前端可通过 `/api/vocabulary` 接口按地区或关键词查询——AI 查询和本地数据库并行，离线时也能查到预置词汇。

## 五、吃货大转盘

**引流逻辑：** 吃货大转盘是留存和社交传播的主力。12 道陕西标志性美食——老潼关肉夹馍、羊肉泡馍、biangbiang 面、葫芦头、凉皮、甑糕等——每道配有实拍图和 50 字古风文化介绍。用户点击"抽"，转盘旋转带咔嗒声效，减速停止后弹出饕餮美食卡。这个体验的传播逻辑是：转盘结果不确定，抽到"冷门美食"用户会好奇，抽到"最爱美食"用户会兴奋——两种情绪都驱动截图分享。美食本身就是天然的社交货币，"看看我抽到了什么陕西美食"比"我用了一个翻译工具"好传播一百倍。随机性让用户愿意重复访问——这次抽到凉皮，下次想试试能不能抽到肉夹馍。

**技术体现：** 转盘使用 CSS `conic-gradient` 绘制 12 个彩色扇区，每个扇区 30°。核心旋转算法：`totalDeg = currentDeg + spins×360 + randomAngle`，通过 `cubic-bezier(0.15,0.8,0.3,1)` 模拟物理惯性和摩擦力减速，总时长 3.5 秒。旋转过程中 `setInterval(SFX.tick, 240)` 每 240ms 播放咔嗒声——Web Audio API Triangle 振荡器 800Hz，40ms 衰减包络。停转后触发 `SFX.gong()` 铜锣声——Sine 振荡器 520→200Hz 指数频率滑降。美食卡弹出采用 `scale(0.9)→1` + `translateY(20px)→0` 的弹簧动画，`cubic-bezier(0.34,1.56,0.64,1)` 制造轻微回弹。

## 六、方言八级考

**引流逻辑：** 考试成绩是社交传播的最强催化剂。方言八级考从 16 道题库中 Fisher-Yates 随机抽取 3 道——每轮题目不同，用户一次拿不到满分就忍不住再来。答对变绿、答错变红的即时反馈让用户在答题过程中持续获得多巴胺刺激。最终成绩分四级——3/3 分"老陕状元"，2/3 分"老陕举人"，1/3 分"还需修炼"，0/3 分"新来的吧"——等级命名用陕西文化梗，增加分享趣味性。"我是老陕状元，来试试你是啥段位"——一张截图就能拉新。16 道题的题库不算大，但正因为小，用户几轮就能覆盖大部分题目，产生"我都见过了"的掌控感，反而更愿意分享成绩。

**技术体现：** Fisher-Yates 洗牌算法保证每轮题目随机且不重复。`pickQuestions()` 每次从题库中无放回抽取 3 道，选项顺序也随机化，降低"记住答案位置"的作弊空间。答案反馈通过动态添加 `.right` / `.wrong` CSS 类实现即时染色，无页面刷新。成绩评定后触发 `saveHistory()` 写入 localStorage 和 SQLite 双持久化，同时更新用户在长安行迹中的时间线记录。题库存储在 `QUESTION_BANK` 数组中，扩展只需追加对象，无需改动评分逻辑。

## 七、长安祈福签

**引流逻辑：** 祈福类内容是所有内容形态中社交分享率最高的——没有之一。八支签文从"上上签"到"下签"分级，签文内容用幽默化解严肃——"上上签：肉夹馍自由，钱包鼓鼓；出门遇贵人，吃啥都不长肉"。用户抽到好签想分享沾喜气，抽到"下签"觉得好笑也想分享吐槽——这两种方向都产生传播。签筒采用实物图片而非 CSS 绘制，配合摇晃抖动动画和签筒声效，增加仪式感和真实感。用户每次想"再摇一次看看运气"就产生一次留存行为，而分享出去的每张截图都是拉新入口。祈福签不需要任何前置知识，男女老少都能玩，是整站门槛最低的引流页面。

**技术体现：** 点击触发 `SFX.rattle()` 签筒摇晃声——6 个 Square 振荡器，频率 300-900Hz 随机分布，间隔 40ms 依次触发，模拟木签在竹筒中碰撞的音效。签筒抖动动画使用 CSS `@keyframes shake`，多层 `translateX` 偏移模拟物理振动。签文从 8 条预设数据中等概率随机抽取，结果通过 `showModal()` 展示。抽签记录通过 `saveHistory()` 持久化，可在长安行迹中回溯。

## 八、评测中心

**引流逻辑：** 评测中心不是引流页，是转化页——它解决"AI 翻译到底准不准"的信任问题。50 条标准语料覆盖 6 大生活场景，每条同步翻译为关中、陕北、陕南三种方言，形成三方言对比柱状图。当用户或潜在合作方（学校、文旅局、博物馆）质疑翻译质量时，这一页用数据而非话术回答。自动评分、三方言横向对比、历史结果累积——长期运行的评测数据本身就是最有说服力的产品背书。

**AI 应用：** 批量评测时，通过 `callAI()` 逐条调用豆包大模型翻译每句语料，每种方言各翻译一次，单轮完整评测产生 150 次 AI 调用。评测结果不依赖人工主观判断，而是采用自建的特征词密度算法自动评分——27 个方言特征词分为关中 12 个、陕北 8 个、陕南 7 个，通过检查翻译输出中特征词的命中率来量化方言"纯正度"。

**技术体现：** 评分算法为 `baseScore=60 + min(40, found×10)`，上限 98 分。评测结果双写至 localStorage 和 SQLite 的 `eval_results` 表，历史数据可通过 `/api/eval/stats` 做聚合统计。柱状图使用 CSS `height` 过渡配合 `cubic-bezier(0.34,1.56,0.64,1)` 弹性动画。支持 10 条（30 次翻译）和 25 条（75 次翻译）两种抽样规模，进度条实时更新。

## 九、长安行迹

**引流逻辑：** 长安行迹是用户的"成就感账本"。它在页面上以时间线形式展示用户在全平台的每一次操作——翻译了什么句子、考了多少分、抽了什么签、查了什么字，按时间倒序排列。这个页面的心理机制是沉没成本：用户看到自己积累了十几条甚至几十条行迹记录，会觉得"我在这里花了不少时间，这是我的专属记录"，离开的心理门槛大幅提高。统计数据区——方言转译次数、古雅言次数、字词解码次数、趣味互动次数——四维量化展示让用户感知到"我不仅是在玩，还在参与方言保护"。时间线本身就是可分享的内容，"看看我的长安行迹"是不同于功能截图的第三种社交传播素材。

**技术体现：** `getHistory()` 从 localStorage 读取全量历史数据，`TYPE_MAP` 将类型码映射为图标和中文标签。每种类型的展示模板不同——翻译类显示"原文→译文"，考试类显示"分数/总分+等级"，转盘类显示"食物名称"。`formatTime()` 将 Unix 时间戳转为 `YYYY-MM-DD HH:MM` 可读格式。历史数据上限 200 条，超出自动裁剪。通过 `/api/history` 可从后端 SQLite 加载更多历史记录。

## 十、方言保护

**引流逻辑：** 这个页面是整个产品的"意义锚点"。它用三组权威数据——陕西省 89 个区县全覆盖、310GB 方言语音档案数据库、10 卷《陕西方言集成》丛书——告诉用户：你在这里的每一次翻译、每一次解码、每一次考试，都不是在"瞎玩"，而是参与了一项有学术根基的方言数字化保护行动。页面底部展示本项目对方言保护的三项贡献——AI 方言转译降低使用门槛、神经网络语音合成留存方言音色、数字化词典构建可检索可传播的知识库——形成"用户在做什么 → 项目在做什么 → 学术在做什麼"的三层意义递进。

**技术体现：** 濒危指示条使用 CSS 渐变宽度柱，关中 65%（safe 翠绿）、陕北 48%（warn 金色）、陕南 42%（warn 金色），数据基于语言学界对方言代际传承的评估。三地方言卡片采用左侧彩色边框区分（关中朱砂红 / 陕北鎏金 / 陕南秘色瓷绿），各卡片内含城市列表和传承状况文字说明。引用块展示《陕西日报》2021 年 11 月 14 日报道原文，增加信息可信度。页面同时展示四项权威资源——《陕西方言集成》《陕西 310GB 语音档案》《中国语言资源保护工程》《中国濒危语言志》。
