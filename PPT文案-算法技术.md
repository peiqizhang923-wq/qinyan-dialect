## 第2页 | 零样本方言翻译方法

**标题：基于角色扮演提示词的大模型方言翻译**

- **核心思路**：不依赖平行语料微调，利用 LLM 对汉语方言的既有知识，通过 System Prompt 激活方言特征
- **三方言差异化策略**：
  - 关中：扮演老西安人，市井口语，"咥""谝闲传""嫽咋咧"
  - 陕北：扮演黄土高原老乡，豪迈质朴，"敞亮""走哇""可美咧"
  - 陕南：扮演陕南居民，轻柔温婉，"蛮舒服""晓得不""莫问题"
- **调用参数**：max_tokens=150（短文本足够），temperature=0.2（低温度保证一致性）
- **效果**：零样本条件下实现三种方言的可用品质翻译

## 第3页 | 方言特征词密度自动评估

**标题：无需人工标注的翻译质量量化方法**

- **算法原理**：统计 AI 翻译结果中方言特征词的出现密度来量化翻译质量
- **评分公式**：基础分 60 + 特征词奖励分（每个特征词 +10，最多 +40）→ 满分 100
- **特征词库**：共 27 个（关中 12 + 陕北 8 + 陕南 7）
  - 例：咥、嫽咋咧、受活、麻达、敞亮、蛮舒服、得行
- **评分等级**：≥85 优 / 70–84 中 / <70 低
- **评测语料库**：6 大场景 × 50 条标准句，Fisher-Yates 随机抽样，单轮可产生 150 个数据点

## 第4页 | 三引擎语音合成与级联容错

**标题：Edge-TTS → 讯飞 → Web Speech 三级降级链路**

- **主力引擎 Edge-TTS**：微软官方陕西话神经语音 zh-CN-shaanxi-XiaoniNeural
  - 五档风格参数（关中+8%语速、陕北-8%、陕南+3%音调、古风-25%）
  - 服务端 3 次重试 + 指数退避（0.8×2^n）应对限流
- **副引擎 讯飞 TTS**：WebSocket 流式合成，纯前端 HMAC-SHA256 鉴权
- **兜底引擎 Web Speech**：浏览器内置，离线可用，rate/pitch 参数模拟三种方言风格
- **透明切换**：任意引擎失败自动降级，用户无感知

## 第5页 | 文本预处理 + HMAC-SHA256 纯前端实现

**标题：提升 TTS 自然度 & 突破 file:// 协议限制**

**文本预处理管线：**
- 语气词后自动插入逗号停顿（咧、呢、嘛、哈等 12 个方言语气词）
- 长句（>18字）在动词/方位词前智能断句
- 前后端双重预处理：JS 端 + Python 端各执行一次，覆盖所有调用路径

**纯前端 HMAC-SHA256：**
- 讯飞 API 要求签名鉴权，浏览器 SubtleCrypto 在 file:// 下不可用
- 纯 JS 实现 SHA-256 + HMAC，约 20 行，零依赖
- 包含完整 UTF-8 多字节编码，中文兼容 Python/CryptoJS
- 使项目可在本地双击 HTML 即运行，无需 Web 服务器
